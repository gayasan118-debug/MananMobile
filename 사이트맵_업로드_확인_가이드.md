# 사이트맵 업로드 확인 가이드

## 문제: sitemap.xml 파일을 찾을 수 없음

브라우저에서 `https://xn--om2b95a99s7bt4be5et2dc4ov7m6nd.com/sitemap.xml`에 접속했을 때 "찾을 수 없다"는 오류가 발생하는 경우, 다음 사항을 확인하세요.

## 확인 사항

### 1. 파일 업로드 확인

**FTP/SFTP 또는 파일 관리자를 통해 확인:**

1. 서버의 **루트 디렉토리**에 다음 파일들이 있는지 확인:
   - `sitemap.xml` ✅
   - `.htaccess` ✅
   - `index.htm` ✅
   - `robots.txt` ✅

2. **파일 위치 확인:**
   - `sitemap.xml`은 반드시 **웹사이트 루트 디렉토리**에 있어야 합니다
   - 예: `/public_html/sitemap.xml` 또는 `/www/sitemap.xml`
   - 하위 폴더에 있으면 안 됩니다

### 2. 파일 권한 확인

**파일 권한 설정:**
- `sitemap.xml`: **644** 또는 **755**
- `.htaccess`: **644** 또는 **755**
- 모든 사용자가 읽을 수 있어야 합니다

**FTP 클라이언트에서 권한 변경 방법:**
1. 파일 선택 → 우클릭 → "파일 권한" 또는 "CHMOD"
2. 권한을 **644**로 설정

### 3. 서버 루트 디렉토리 확인

**확인 방법:**
1. `index.htm` 파일이 접속되는지 확인
   - `https://xn--om2b95a99s7bt4be5et2dc4ov7m6nd.com/` 접속
   - 정상적으로 표시되면 그 디렉토리가 루트입니다

2. `robots.txt` 파일이 접속되는지 확인
   - `https://xn--om2b95a99s7bt4be5et2dc4ov7m6nd.com/robots.txt` 접속
   - 정상적으로 표시되면 그 디렉토리가 루트입니다

3. `sitemap.xml`도 같은 디렉토리에 있어야 합니다

### 4. .htaccess 파일 확인

**.htaccess 파일이 올바르게 업로드되었는지 확인:**
- 파일명이 정확히 `.htaccess`인지 확인 (앞에 점(.)이 있어야 함)
- 일부 FTP 클라이언트는 숨김 파일을 표시하지 않을 수 있습니다
- "숨김 파일 표시" 옵션을 활성화하세요

### 5. 서버 설정 확인

**일부 호스팅 환경에서는:**
- `.htaccess` 파일이 비활성화되어 있을 수 있습니다
- 서버 관리자에게 문의하여 `.htaccess` 사용 가능 여부 확인

## 해결 방법

### 방법 1: 파일 재업로드

1. **로컬 파일 확인:**
   ```
   D:\만안중앙하이츠\만안역중앙하이츠포레.com\Icheon-Shinan-Insvill\sitemap.xml
   ```

2. **FTP/SFTP로 서버에 업로드:**
   - 서버 루트 디렉토리에 업로드
   - ASCII 모드로 업로드 (바이너리 모드 아님)
   - 파일 권한을 644로 설정

3. **업로드 후 확인:**
   - 브라우저에서 `https://xn--om2b95a99s7bt4be5et2dc4ov7m6nd.com/sitemap.xml` 접속
   - XML 형식으로 표시되어야 합니다

### 방법 2: 서버 관리자에게 문의

다음 정보를 서버 관리자에게 제공하세요:

```
문제: sitemap.xml 파일에 접근할 수 없습니다.
URL: https://xn--om2b95a99s7bt4be5et2dc4ov7m6nd.com/sitemap.xml
요청사항:
1. sitemap.xml 파일이 서버 루트에 있는지 확인
2. 파일 권한이 올바른지 확인 (644 또는 755)
3. .htaccess 파일이 작동하는지 확인
4. XML 파일의 MIME 타입이 application/xml로 설정되어 있는지 확인
```

### 방법 3: 대체 방법 (IIS 서버인 경우)

IIS 서버를 사용하는 경우, `.htaccess` 대신 `web.config` 파일을 사용해야 합니다.

**web.config 파일 생성:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
  <system.webServer>
    <staticContent>
      <mimeMap fileExtension=".xml" mimeType="application/xml" />
    </staticContent>
  </system.webServer>
</configuration>
```

## 테스트 방법

### 1. 브라우저에서 직접 접속
```
https://xn--om2b95a99s7bt4be5et2dc4ov7m6nd.com/sitemap.xml
```

**정상적인 경우:**
- XML 형식으로 표시됨
- 브라우저가 XML을 파싱하여 보기 좋게 표시

**비정상적인 경우:**
- 404 오류: 파일이 없거나 경로가 잘못됨
- HTML로 표시됨: MIME 타입 문제
- 빈 페이지: 파일 권한 문제

### 2. 개발자 도구로 확인

1. 브라우저에서 F12 키 누르기
2. Network 탭 열기
3. `sitemap.xml` 접속
4. 응답 헤더 확인:
   - `Content-Type: application/xml` 또는 `text/xml`이어야 함
   - `Content-Type: text/html`이면 문제가 있음

### 3. curl 명령어로 확인 (터미널/명령 프롬프트)

```bash
curl -I https://xn--om2b95a99s7bt4be5et2dc4ov7m6nd.com/sitemap.xml
```

**정상적인 응답:**
```
HTTP/1.1 200 OK
Content-Type: application/xml; charset=UTF-8
```

## 체크리스트

업로드 전 확인:
- [ ] `sitemap.xml` 파일이 로컬에 있음
- [ ] `.htaccess` 파일이 로컬에 있음
- [ ] 파일 내용이 올바름

업로드 후 확인:
- [ ] 서버 루트 디렉토리에 파일이 있음
- [ ] 파일 권한이 644 또는 755로 설정됨
- [ ] 브라우저에서 접속 가능함
- [ ] XML 형식으로 표시됨
- [ ] Content-Type이 application/xml임

## 추가 도움말

여전히 문제가 해결되지 않으면:
1. 서버 로그 확인 (에러 로그)
2. 호스팅 제공업체 고객 지원에 문의
3. 다른 XML 파일(예: rss.xml)이 접근 가능한지 확인하여 비교
